### Writing Style

The writeups are written in a practical and human style.

I try to explain what I was thinking at each step instead of only showing the final exploit.

A typical writeup includes:

```
- What the page looked like at first.
- What endpoints or parameters looked interesting.
- What payloads I tried.
- What failed.
- What worked.
- Why the bug existed.
- How I automated it.
- How the vulnerability could be patched.
```

The goal is to make each writeup useful for learning, not just for copying flags.
